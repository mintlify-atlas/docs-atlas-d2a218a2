OpenClaude is a CLI coding agent — a fork of Claude Code — that works with any OpenAI-compatible model provider. This guide walks you through installation, provider setup, and your first run on macOS or Linux.

<Steps>

<Step title="Install Node.js 20 or newer">

OpenClaude requires Node.js 20 or newer. Download the LTS release from [nodejs.org](https://nodejs.org/) and follow the installer.

Verify your install:

```bash
node --version
npm --version
```

Both commands should return a version number. If `node` is not found, restart your terminal after the installer finishes.

</Step>

<Step title="Install OpenClaude">

Install the `@gitlawb/openclaude` package globally:

```bash
npm install -g @gitlawb/openclaude
```

The package installs a single binary named `openclaude`. Confirm it is available:

```bash
openclaude --version
```

<Note>
If `openclaude` is not found immediately after install, close the terminal and open a new one. npm's global bin directory may not be on your `PATH` in the current session.
</Note>

</Step>

<Step title="Set your provider environment variables">

OpenClaude uses environment variables to know which LLM backend to call. Pick one provider to start with — you can switch later.

<Tabs>
  <Tab title="OpenAI">
    Replace `sk-your-key-here` with your real key from [platform.openai.com](https://platform.openai.com/api-keys).

    ```bash
    export CLAUDE_CODE_USE_OPENAI=1
    export OPENAI_API_KEY=sk-your-key-here
    export OPENAI_MODEL=gpt-4o
    ```

    `gpt-4o` has excellent tool-calling support and is the recommended default for new users.
  </Tab>
  <Tab title="DeepSeek">
    Get your key from [platform.deepseek.com](https://platform.deepseek.com/).

    ```bash
    export CLAUDE_CODE_USE_OPENAI=1
    export OPENAI_API_KEY=sk-your-key-here
    export OPENAI_BASE_URL=https://api.deepseek.com/v1
    export OPENAI_MODEL=deepseek-chat
    ```

    DeepSeek uses an OpenAI-compatible API. Setting `OPENAI_BASE_URL` points the shim at DeepSeek's endpoint instead of OpenAI's.
  </Tab>
  <Tab title="Ollama (local)">
    Ollama runs models locally — no API key required.

    First, install Ollama from [ollama.com/download](https://ollama.com/download), then pull a model:

    ```bash
    ollama pull llama3.1:8b
    ```

    Then set the variables:

    ```bash
    export CLAUDE_CODE_USE_OPENAI=1
    export OPENAI_BASE_URL=http://localhost:11434/v1
    export OPENAI_MODEL=llama3.1:8b
    ```

    <Note>
    No `OPENAI_API_KEY` is needed for Ollama. Make sure Ollama is running before you start OpenClaude — it does not auto-start.
    </Note>
  </Tab>
</Tabs>

</Step>

<Step title="Run OpenClaude">

Navigate to a project directory and launch the CLI:

```bash
cd /your/project
openclaude
```

The CLI starts and waits for your prompt. You can ask it to explain code, edit files, run commands, review changes, or commit work — all tools from Claude Code work the same way.

</Step>

<Step title="Make variables permanent">

Environment variables set with `export` only last for the current shell session. To avoid re-setting them every time, append them to your shell profile.

<Tabs>
  <Tab title="zsh (~/.zshrc)">
    ```bash
    echo 'export CLAUDE_CODE_USE_OPENAI=1' >> ~/.zshrc
    echo 'export OPENAI_API_KEY=sk-your-key-here' >> ~/.zshrc
    echo 'export OPENAI_MODEL=gpt-4o' >> ~/.zshrc
    source ~/.zshrc
    ```
  </Tab>
  <Tab title="bash (~/.bashrc)">
    ```bash
    echo 'export CLAUDE_CODE_USE_OPENAI=1' >> ~/.bashrc
    echo 'export OPENAI_API_KEY=sk-your-key-here' >> ~/.bashrc
    echo 'export OPENAI_MODEL=gpt-4o' >> ~/.bashrc
    source ~/.bashrc
    ```
  </Tab>
</Tabs>

<Warning>
Replace `sk-your-key-here` with your actual API key before running these commands. Storing a placeholder in your profile will cause authentication errors on every new terminal session.
</Warning>

</Step>

</Steps>

## Update OpenClaude

To update to the latest version:

```bash
npm install -g @gitlawb/openclaude@latest
```

## Uninstall

```bash
npm uninstall -g @gitlawb/openclaude
```

## Troubleshooting

### `openclaude` command not found after install

npm installs global binaries into a directory that may not be on your `PATH`. Close the terminal completely, open a new one, and run `openclaude` again. If it still fails, run `npm bin -g` to find the global bin directory and add it to your `PATH` in your shell profile.

### Invalid API key error

The key you set in `OPENAI_API_KEY` was rejected. Check that:

- You copied the full key — keys are long and easy to truncate
- The key is active in your provider's dashboard
- You are not using a key from a different provider (e.g., an Anthropic key against the OpenAI endpoint)

Re-export the corrected key and run `openclaude` again.

### Ollama: connection refused

OpenClaude cannot reach `http://localhost:11434/v1`. Make sure:

- Ollama is installed — run `ollama --version`
- Ollama is running — start it with `ollama serve` or from the Ollama app
- Your chosen model was pulled successfully — run `ollama list` to confirm

### `ripgrep not found`

OpenClaude uses `rg` for the Grep tool. Install ripgrep system-wide:

```bash
# macOS (Homebrew)
brew install ripgrep

# Ubuntu / Debian
sudo apt install ripgrep

# Fedora
sudo dnf install ripgrep
```

Confirm it works with `rg --version`, then restart OpenClaude.
