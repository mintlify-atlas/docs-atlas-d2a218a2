title: Tools
description: All tools available in the OpenClaude agent loop, including file, shell, web, task, MCP, and utility tools.

OpenClaude provides the same tool set as the original Claude Code. Every tool works through the OpenAI shim with any provider that supports function calling.

<Note>
  For best results, use models with strong function/tool calling support. Smaller models (under 7B parameters) may struggle with multi-step tool chains. See the [model quality guide](/troubleshooting/model-quality) for recommendations.
</Note>

## File tools

| Tool | Source directory | Description |
|------|-----------------|-------------|
| `FileRead` | `src/tools/FileReadTool/` | Read a file or list a directory |
| `FileWrite` | `src/tools/FileWriteTool/` | Write content to a file (creates or overwrites) |
| `FileEdit` | `src/tools/FileEditTool/` | Apply exact string replacements in a file |
| `Glob` | `src/tools/GlobTool/` | Find files by glob pattern |
| `Grep` | `src/tools/GrepTool/` | Search file contents with regular expressions |
| `NotebookEdit` | `src/tools/NotebookEditTool/` | Edit Jupyter notebook cells |

## Shell tools

| Tool | Source directory | Description |
|------|-----------------|-------------|
| `Bash` | `src/tools/BashTool/` | Run a shell command in a persistent session |
| `PowerShell` | `src/tools/PowerShellTool/` | Run a PowerShell command (Windows) |

## Web tools

| Tool | Source directory | Description |
|------|-----------------|-------------|
| `WebFetch` | `src/tools/WebFetchTool/` | Fetch a URL and convert to markdown |
| `WebSearch` | `src/tools/WebSearchTool/` | Search the web |

<Warning>
  `WebSearch` is disabled for non-Anthropic providers by default. Set `FIRECRAWL_API_KEY` to enable it for any provider. Without the key, `WebFetch` also falls back to basic HTTP and will fail on JavaScript-rendered pages.
</Warning>

## Agent and task tools

| Tool | Source directory | Description |
|------|-----------------|-------------|
| `Agent` | `src/tools/AgentTool/` | Spawn a sub-agent to handle a subtask |
| `TaskCreate` | `src/tools/TaskCreateTool/` | Create a background task |
| `TaskGet` | `src/tools/TaskGetTool/` | Get the status or output of a task by ID |
| `TaskList` | `src/tools/TaskListTool/` | List all tasks |
| `TaskUpdate` | `src/tools/TaskUpdateTool/` | Update a task |
| `TaskStop` | `src/tools/TaskStopTool/` | Stop a running task |
| `TaskOutput` | `src/tools/TaskOutputTool/` | Retrieve task output |

Sub-agents spawned by `Agent` use the same provider and model as the parent session. They share no state with the parent except what is passed explicitly in the task description.
